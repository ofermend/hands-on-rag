The book

From a RAG demo to a system you can run in production

The market is full of RAG pipelines, frameworks, and vector databases, which makes it hard to know what to actually use. This book walks through every phase of building a RAG application — data ingestion, embeddings, and vector search through to advanced techniques like agentic RAG, multimodal RAG, and GraphRAG — and is honest about the trade-offs at each step.

The focus is production: managing hallucinations, keeping latency and cost in check, securing your data, and producing answers you can explain. It also helps you make the call that matters most — whether to build a RAG system yourself or stand on a managed platform.

Build the base stack

Parsing, chunking, embeddings, vector search, and generation — the components every RAG system is made of.

Scale retrieval quality

Reranking, guardrails, and hallucination detection to keep answers grounded as the corpus grows.

Ship to production

Cut latency and cost with caching, protect sensitive data with redaction, and deploy with confidence.

Evaluate honestly

LLM-as-a-judge and the metrics that actually tell you whether your system is getting better.

Build vs. platform

A clear-eyed look at when a managed RAG platform earns its keep and when to roll your own.

Go beyond text

Agentic RAG, multimodal RAG over images and audio, and knowledge-graph RAG for harder questions.

Run it yourself

Every example, as a notebook

The book ships with a companion repository of Jupyter notebooks — one set per chapter, runnable locally or in Google Colab. They use LangChain, LlamaIndex, and CrewAI alongside pgvector, SQLite-vec, Neo4j, and models from OpenAI and Anthropic, so you can follow along and adapt the code to your own data.
